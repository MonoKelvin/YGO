## 首要规则

- **每次AI对话**：不要还原之前手动修改的内容。
- **AI删除文件、代码时**：删除文件时需确认其他地方是否有引用。如删除代码片段，连同html、js、css等关联代码都需删除，除非有引用。
- **代码管理**：没有明确说明提交并推送到git远程仓库就不要自行提交代码。有重大功能修改内容或者变更等要同步更新 `README.md` 文件。

## 编码设计

- **代码风格**：使用 **2 个空格**进行缩进，不要使用 Tab 字符。在代码中添加必要的注释，包括函数、类、组件等。
- **组件职责单一和复用**：尽量复用软件提供的通用组件，如果新增的组件可能通用也需要提取出来作为通用组件使用，大的页面可以拆分多个组件分文件编码，业务和UI分离。
- **代码质量**：编码完成后要检查是否逻辑、编码问题，确保功能正确无误易读且容易维护。

## UI 设计规范

- **布局**：采用响应式布局，支持不同屏幕尺寸，合理设计对齐、间距、填充等。注意：内容应该精简，减少重复描述的文字、标题或者无意义的内容，比如大标题已经有文字描述了，就不需要再每个工具模块重复添加。
- **字体**：使用系统默认字体，最小的字体应该使用`-md`结尾的字体大小变量，除非有必要用更小的字体。
- **风格**：高级简约，现代化 UI 设计，统一样式，使用代码中定义的 CSS 变量和主题变量。需要支持深浅模式。动画尽可能使用非线性动画，保证流畅不冗余。
- **图标**：优先使用三方库提供的图标，再复用已有 SVG 图标，新增图标放到 `assets/icons` 目录。

## 性能优化

- **懒加载**：按需加载。加载大量数据时，避免一次性加载所有数据，滚动到底部时再加载更多数据。
- **异步和多线程**：处理大量、复杂数据时，采用异步和多线程，如需要频繁刷新的数据离开页面时停止刷新，进入页面时异步加载数据，不卡界面，使用正在加载等描述占位。
